css는 작업이 고도화 되거나 사이즈가 커지면 굉장히 불편함이 많아진다. 때문에, css preprocessor인 sass, less, stylus와 같은 것들이 생긴 것 이다. css를 보완하는 개념이라고 생각하자. 하지만 웹에서는 표준 css만 동작 할 수 있기 때문에 sass를 그대로 쓰는것은 불가능하다. 1. 문법을 쉽게 작성하기 위해 css preprocessor인 sass를 통해 작성한다. 2. 작성된 내용을 컴파일(compile) 하여, css문법으로 전환시킨다. css의 문법을 편하고 쉽게 작성하기 위함으로, css구동 전단계에 해당 하므로 'css전처리기'라 불린다. 보편적으로 가장 많이 사용하는 css preprocessor는 SASS혹은 SCSS 라 불리는 전처리기이다. 이유는, less나 stylus보다 먼저 나와 성숙하다. 문제점 발견시 해결방법이 더 많다. 등등
