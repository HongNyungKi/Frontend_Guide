@@@ 웹에서 사용하는 이미지 @@@ 이미지(그래픽)는 크게 비트맵과 벡터로 구분된다. 비트맵(bitmap): 은 각 픽셀이 모여 만들어진 정보의 집합으로 레스터(raster)이미지라고도 부른다. 픽셀단위로 화면에 렌더링(컴퓨터가 화면에 그림을 그려서 볼 수 있게 하는 것)한다. 일반적으로 사용하는 대부분의 이미지가 비트맵 형식이다. 그림판, 포토샵과 같으느 툴로 편집 할 수 있다. ex) jpeg / gif / png 벡터(vector): 는 수학적 정보의 형태(shape)들이 만들어내는 결과물이다. 이미지가 갖고 있는 점,선,면의 위치(좌표)와 색상 등의 정보를 온전히 갖고 그를 화면에 렌더링한다. 때문에 축소,확대 시 깨지지 않고, 용략의 변화가 없다. 일러스트 같으느 툴로 편집 할 수 있다. ex) svg - JPG(JPEG) 높은 압축률, 적은 용량!!! 1. 손실압축방식이다 때문에 여러번 압축시 용량은 적게 들지만 품질이 저하된다. 2. 표현 색상도(24비트,1600만 색상)가 뛰어나 고해상도 표시에 적합하다. 3. 이미지의 품질과 용량을 쉽게 조절 가능하다. 4. 가장 널리 쓰인다. - PNG 투명도 지원!!! 1. 비손실 압축방식이다. 2. 8비트(256색상) / 24비트(1600만 색상) 컬러 이미지 처리. 3. 투명도 지원 4. W3C권장 포맷 - GIF 동영상 같은 이미지!!!(애니메이션) 1. 비손실 압축방식이다. 2. 여러장의 이미지를 한개의 파일에 담을 수 있다.(움짤,애니메이션) 3. 8비트 컬러만 지원(다양한 색상을 표현하는 작업에는 부적합) - WEBP 완벽한 포맷! 그러나 지원 브라우저가 제한되어 있다. 1. jpg,png,gif를 모두 대체할 수 있는 구글이 개발한 이미지 포맷이다. 2. 완벽한 손실,비손실 압축 지원 3. gif 같은 애니메이션 지원 4. alpha channel(투명도)지원(손실,비손실 모두) - SVG svg는 마크업 언어(html/xml)기반의 벡터 그래픽을 표현하는 포맷이다. 1. 해상도의 영향에서 자유롭다. 2. css로 styling 가능. 3. javascrpt로 event handling가능. 4. 코드 혹은 파일로 사용 가능.
